1. Introduction¶

1.1 Background¶

Traffic accidents are considered as a significant cause of deaths, injuries, and property damage. It has been a main concern to the National Safety Council. Moreover, these accidents are affecting other individuals who are not mainly involved in the accident by delaying their trip and corrupting their plans. Having an understanding of what causes car accidents is the secret to avoid being involved in one. In other words, knowing how bad an accident can be based on certain identifiers can help individuals avoid suffering the consequences of being involved in them, by driving more carefully if came across these types of identifiers or even change their plans. Therefore, it became important to predict the severity of accidents based on their characteristics like if this accident will cause death fatalities or traffic delay, or any other bad impact of accidents.

1.2 Problem¶

Data that might contribute to determining the severity of each accident might include the characteristics that describe each accident like the type of collision, location, how many cars were involved in the accident, how many people were involved in the accident, weather condition, road condition, and some other characteristics that describe the causes of the collision. This project aims to predict the severity of accidents based on these data.

1.3 Interest¶

Obviously, drivers would be very interested in accurate prediction of accidents bad impacts so they can be proactive with regard to these undesired situations.

2. Data acquisition and cleaning¶

2.1 Data source¶

The dataset that will be used for building the predicting model is taken from the Seattle Police Department and it has all collisions that were recorded from 2004 to present. The dataset can be found in the following link: https://s3.us.cloud-object-storage.appdomain.cloud/cf-courses-data/CognitiveClass/DP0701EN/version-2/Data-Collisions.csv

2.2 Data cleaning¶

Data is downloaded from only one source. It has 38 columns and 194,673 records. The problems that I have identified within this dataset are as follow: First, I changed the data type for the accident date and the accident date/time from String to Date/Time. Second, there is an inconsistency in the values of the UNDERINFL column, this column tells whether or not a driver involved was under the influence of drugs or alcohol. The actual values have is either Yes or No, but we have 0 and 1 as well. Practically 0 represents No and 1 represent Yes, so I unified the representation for the data to the character representation. The third issue is the percentage of Null values within the dataset. Out of the current 38 columns, there are 18 features that contain Null values with percentages that vary from 0.009% to 97%. I dropped the columns that are unpredictable and the empty rows from the rest of the columns. The fourth issue that I faced, is that there is some inconsistency that I found in the relationship between the Accident time and the light condition, and the WEATHER and ROADCOND columns. I fixed these data inconsistencies and now my dataset is ready for analysis.

2.3 Feature selection¶

After cleaning data, there were 180,067 samples and 28 features in the data. Upon examining the meaning of each feature, out of the 35 features, the below list of features will be selected for building the predicting model,

  • WEATHER
  • ADDRTYPE
  • COLLISIONTYPE
  • UNDERINFL
  • ROADCOND
  • LIGHTCOND
  • INCDTTM
  • PERSONCOUNT
  • VEHCOUNT

    4. Predictive Modeling¶

    In this research we will use the classification model to predict to which category an accident might classified as based on the charactaristics that we have analyzed.

    4.1 Classification models¶

    In the application of classification models, I started by normalizing the data and then divided it into two sets, the training and the testing sets. I built K-nearesr neighbor, logistic regression, SVM, and decision tree models. Among the individual models, the SVM model performed the best with ~73% accuracy. Logistic regression model performed similarly as the SVM model, though the differences between models were small. Below is a table with the accuracy of each algorithm that was used to build the model.

    5. Conclusions¶

    In this study, I analyzed the relationship between the severity of car accidents and traffic accident characteristics. I identified address type, collision type, weather, and people count to be the most influencing features of the accident severity. I built a classification model to predict whether an accident will be an injury or property damage accident. These models can be very useful in helping individuals in a number of ways. For example, it could help identify which days to set plans for and when to drive more carefully, etc.
